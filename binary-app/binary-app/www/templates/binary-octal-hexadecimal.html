What other number systems do computer scientists use?

Alright, so we have conversion between decimal and binary down. What's next? Well, computer scientists also use octal (base8) and hexadecimal (base16) systems fairly frequently. In fact, the characters you see on this screen are actually represented in code using hexadecimal!

"But wait!" you say. You can probably guess that base8 uses the digits 0 - 7, but what about base16? Does it use the "digits" 0 - 15? Unfortunately, no. The "digits" 10 - 15 are not actually standalone digits, they are a combination of previous digits and therefore can't be used as they are. Instead, we represent the "digits" 10 - 15 using the characters A - F, respectively. This means that base16 uses the digits 0 - 9 and A - F.

"Okay", you say, "But why bother doing this at all? It just looks like extra work."

Well, one reason most people prefer to convert base8 and base16 numbers into base2 first is because base2 is easier to compute. If you know any given power of 2 (say, 24 = 16), then you can easily figure out what the adjacent powers are by dividing or multiplying by 2 (23 = 16/2 = 8; 25 = 16 * 2 = 32). Base8 and base16 are a little harder, especially when you get into the higher powers. Who wants to spend their time computing 165? And once you compute 165, finding 164 and 166 is a little harder than it was in base2; now you have to divide and multiply by 16. Another reason the two-step process is preferred is because most computer scientists become familiar with binary anyway, so we might as well use the knowledge we already have to make things easier, right?

This process also works in reverse. If you take our old base2 number, 10010110, you can turn it into base8 by grouping every three digits together, or base16 by grouping every four digits together. However, it is very important that you start grouping from the right. This means that you might need to add preceding zeroes to the left of the binary number in order to make the right size of grouping.

To convert to base8, we have to make groups of three. If we start grouping from the right side, we get the following:
10|010|110
But we have to make sure that each of these groups have three digits, so we add a 0 to the front like so:
010|010|110
We can then convert each group into a single base8 digit. The table below should help you visualize this.

We already know that 10010110 in base10 is 150, but now we can see that in base8 it's 226 and in base16 it's 96. Again, the two-step conversion is to help you reduce the number of powers you need to remember. Why bother memorizing the powers of 16 and 8 when you can just stick with the powers of 2?

Our last example will hopefully drive the process home. Say we want to convert the base10 number 4,792 into both base8 and base16. First, let's do it directly.

Base16:
4,792 - (1 * 163) = 696
696 - (2 * 162) = 184
184 - (11 * 161) = 8
8 - (8 * 160) = 0

Which is the same as:

4,792 - 4,096 = 696
696 - 512 = 184
184 - 176 = 8
8 - 8 = 0

Therefore...

4,79610 = 12B816

Notice how, at each power that was used, we multiplied the power by the number of times we could use it. For example, 696 is too small to subtract another power of 3 (163 = 4,096), but we can definitely subtract a power of 2 from it (162 = 256). In fact, we can subtract 256 twice from 696 before it becomes too small. We do the same with 184 and subtracting (11 * 16) from it. We then take the number of times we we used each power and string them together to create the base16 number 12B8 (remember, B = "11" in base16).

Now, doing the same thing with base8...

Base8:
4,792 - (1 * 84) = 696
696 - (1 * 83) = 184
184 - (2 * 82) = 56
56 - (7 * 81) = 0

Which is the same as:

4,792 - 4,096 = 696
696 - 512 = 184
184 - 128 = 56
56 - 56 = 0

Therefore...

4,79610 = 11,2708

Notice this time that we had to add a 0 to the end because we used 80 0 times.

Remember, in order to go to base2 from here we have to take each power that was used and set that position = 1 and all other positions = 0. Once we do that, we get:
4,79210 = 10010101110002
