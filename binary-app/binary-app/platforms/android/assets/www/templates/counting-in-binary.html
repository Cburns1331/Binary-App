How do you count in binary?

Counting in binary isn't hard, but it takes a little getting used to since it is different from base10. In base10 (what we humans normally use), we count from 0 to 9 because those are the digits we have. As we all know, the next number is 10. We increase the left digit by one and send the right digit back to the "beginning". A good way to see this in action is when you write 0 - 9 in the following way.

0 = 00
1 = 01
2 = 02
...
9 = 09

9 is the highest single digit possible in base10 (remember, base10 uses the digits 0 - 9). So, when we want to get one more than 9, we increment the left digit (which has been 0 this whole time) by one, and we return the right digit to 0 and begin counting up to 9 again.

10
11
12
...

This incrementing scheme can be seen again with 100:
001, 002, 003, ..., 097, 098, 099, 100, 101, 102
As we can see from these examples, counting in base10 is just incrementing the right digit to the maximum value it can be and then incrementing the left digit once and resetting the right digit to the minimum value it can be. Base10 counts using the digits 0 - 9 and base2 counts using the digits 0 - 1. So how do you think we count in base2? I'll show you the base 10 numbers for comparison.
