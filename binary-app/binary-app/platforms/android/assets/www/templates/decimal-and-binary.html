How do you convert between a decimal number and a binary number?

Alright, so you understand how to count in binary. That wasn't too hard, right? But conceptually, what's going on here?

In base10, as we add digits to the left, we're increasing by a magnitude of 10. So, for the number 15, the 1 digit is the equivalent of 10 to the power of 1. The 5 digit is the equivalent of 10 to the power of 0 (remember, any digit to the power of 0 equals 1). We then take those digits and multiply them by their powers.

You should see a pattern emerging. The rightmost digit is considered as "position 0". The next-left digit is "position 1", the next-left from there is "position 2", to the left of that digit is "position 3", and so forth. 10010110: How can you figure out what this number is supposed to be when converted to base10? You could count all the way up to that number in binary if you really wanted to, but it would take a pretty long time. Instead, we can use the (digit * baseposition) formula from our previous examples to convert any baseX to a base10 number. This means the binary (base2) number 10010110 is actually the decimal (base10) number 150.

What about converting the other way, from decimal to binary? It's simple--we just reverse the process we've used so far. We can start with a decimal number, like 150, and find the highest power of 2 that is less than that number. For example, 28 is 256, which is higher than 150 so we can't use it. But 27 is 128, which is less than 150 so we can use it. We continue this pattern until we get to 0.

150 - 128 (27) = 22 - 16 (24) = 6 - 4 (22) = 2 - 2 (21) = 0
